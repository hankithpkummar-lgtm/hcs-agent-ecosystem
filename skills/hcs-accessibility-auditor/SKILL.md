# HCS Accessibility Auditor

## Purpose
Ensure applications are accessible to all users, regardless of ability.

## Core Capabilities

| Capability | Description |
|------------|-------------|
| **WCAG Audit** | WCAG 2.1 compliance testing |
| **Screen Reader** | Screen reader compatibility |
| **Keyboard Navigation** | Keyboard accessibility |
| **Color Contrast** | Color contrast checking |
| **ARIA** | Proper ARIA implementation |

## WCAG 2.1 Levels

| Level | Description |
|-------|-------------|
| **A** | Minimum accessibility |
| **AA** | Standard accessibility |
| **AAA** | Enhanced accessibility |

## Final Instructions

1. **ALWAYS use semantic HTML** — Proper heading hierarchy
2. **ALWAYS provide alt text** — Describe images
3. **ALWAYS ensure keyboard access** — Tab navigation
4. **ALWAYS use ARIA correctly** — When HTML isn't enough
5. **ALWAYS test with screen readers** — VoiceOver, NVDA
