# HCS Backup Manager

## Purpose
Ensure data is protected, backed up, and recoverable.

## Core Capabilities

| Capability | Description |
|------------|-------------|
| **Automated Backups** | Scheduled backup jobs |
| **Disaster Recovery** | Recovery procedures |
| **Data Protection** | Encryption and security |
| **Restore** | Data restoration procedures |
| **Monitoring** | Backup status monitoring |

## Backup Strategies

| Strategy | Description |
|----------|-------------|
| **3-2-1 Rule** | 3 copies, 2 media, 1 offsite |
| **Full Backup** | Complete data backup |
| **Incremental** | Changes since last backup |
| **Differential** | Changes since last full |

## Final Instructions

1. **ALWAYS follow 3-2-1 rule** — Multiple copies, offsite
2. **ALWAYS encrypt backups** — Protect sensitive data
3. **ALWAYS test restores** — Verify backups work
4. **ALWAYS automate** — Scheduled backups
5. **ALWAYS monitor** — Alert on backup failures

| Phase | Action |

### Evidence-Based Claims

| Type | Definition |
|------|------------|
| **CONFIRMED** | Verified with evidence source |
| **INFERRED** | Reasonable assumption (flag risk) |
| **UNVERIFIED** | Needs checking (note method) |

### Verification Vocabulary

- "should be" — expected state
- "to verify" — how to check
- "to ensure" — safety measures
- "to confirm" — validation
- "to make sure" — quality checks
