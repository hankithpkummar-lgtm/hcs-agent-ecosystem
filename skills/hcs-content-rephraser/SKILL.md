# HCS Content Rephraser

## Purpose
Transform ordinary content into exceptional, well-researched, comprehensive reports through the structured **Analyze → Research → Implement → Rephrase** pipeline.

## Core Capabilities

| Capability | Description |
|------------|-------------|
| **Analyze** | Analyze existing code/content comprehensively |
| **Research** | Deep research on ALL topics identified |
| **Implement** | Prepare implementation plan for new topics |
| **Rephrase** | Rephrase content according to website type |
| **Enhance** | Improve quality, tone, and professionalism |
| **Expand** | Add details, examples, statistics |
| **Cite** | Add proper source citations |
| **Report** | Generate comprehensive reports |

## Rephrasing Techniques

### Sentence Structure

| Technique | Example |
|-----------|---------|
| **Active Voice** | "Team completed" vs "Project was completed" |
| **Varied Length** | Mix short and long sentences |
| **Parallel Structure** | Consistent grammar in lists |
| **Remove Wordiness** | "Due to" → "Because" |

### Vocabulary Enhancement

| Basic → Professional | Basic → Technical |
|---------------------|-------------------|
| "good" → "excellent" | "use" → "utilize" |
| "show" → "demonstrate" | "make" → "generate" |
| "help" → "facilitate" | "get" → "obtain" |
| "big" → "significant" | "use" → "leverage" |

## Final Instructions

1. **ALWAYS follow the pipeline** — Analyze → Research → Implement → Rephrase
2. **NEVER skip analysis** — Understand existing content first
3. **NEVER skip research** — Always research before enhancing
4. **NEVER fabricate data** — Use real sources only
5. **NEVER lose original meaning** — Enhance, don't change
6. **ALWAYS cite sources** — Proper attribution
7. **ALWAYS expand meaningfully** — Add value, not filler
8. **ALWAYS adapt to website type** — Match tone, vocabulary, style
9. **ALWAYS create implementation plan** — Document new topics to add
10. **ALWAYS integrate research** — Weave findings throughout content

**This skill follows the Fabel5 six-phase senior-engineer loop.**

### Core Principle: THE MAKER IS NEVER THE GRADER

| Phase | Action |
|-------|--------|
| **1. THINK** | Map system, find source of truth, diagnose from real artifact |
| **2. DECOMPOSE** | Split into ONE bounded units, parallel where possible |
| **3. PROVE IT** | Build, run tests, validate — never claim without evidence |
| **4. RESPECT INTENT** | Never reverse decisions silently, surface recommendations |
| **5. VERIFY DELIVERY** | Confirm change landed, skeptic pass, fix critical issues first |
| **6. LEAVE NAVIGABLE** | Update notes, codify rules, write STATE.md |

### Evidence-Based Claims

| Type | Definition |
|------|------------|
| **CONFIRMED** | Verified with evidence source |
| **INFERRED** | Reasonable assumption (flag risk) |
| **UNVERIFIED** | Needs checking (note method) |

### Verification Vocabulary

- "should be" — expected state
- "to verify" — how to check
- "to ensure" — safety measures
- "to confirm" — validation
- "to make sure" — quality checks

### Final Instructions

1. Be skeptical — Find issues, dont confirm everything
2. Be thorough — Check every claim
3. Be honest — Say clearly if wrong
4. Be specific — Provide exact findings
5. Be constructive — Suggest fixes
